# Changes applied on 2026-04-15

Summary of safe improvements applied from `plan.md` to make the project more robust and safer.

1) .gitignore updated
- Added entries to ignore `models/`, `*.pkl`, `*.joblib`, `*.db`, `*.sqlite` so persisted models and databases are NOT committed.

2) requirements.txt updated
- Added `joblib>=1.2.0` to support model persistence and safe load/save helpers.

3) Added `model_io.py`
- Provides `save_model`, `load_model`, and `get_model_hash` wrappers around `joblib` with:
  - Try/except error handling to avoid crashes on corrupted files
  - SHA256 digest computation for optional verification
  - Small docs and usage notes (warning about pickle safety)

Why these changes?
- Prevent accidental commits of large or sensitive artifacts (models, local DBs).
- Provide a safe, defensive API to persist and restore models without crashing the app if files are corrupted or tampered with.
- Require `joblib` explicitly so environments can be provisioned reliably.

Notes & Next Steps (recommended):
- Integrate `model_io` into your `predictor.py` or `model_manager.py` to save/load models via `save_model`/`load_model`.
- Consider storing the model hash alongside metadata when saving, then verify on load.
- Add a brief PII redaction step before `save_data()` if you ever start saving contact info.
- Run a `pip-audit` or `safety` check on your dependencies before publishing.
